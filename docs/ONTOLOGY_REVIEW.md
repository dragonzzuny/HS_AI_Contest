# 온톨로지 AX 검증·리뷰·개선 보고

> 요청: "온톨로지를 AX(AI eXperience) 관점에서 진지하게 검증·리뷰·개선."
> 결론: 초기 구현은 **표면적(decorative)**이었고 일부는 **헛링크/쓰레기 데이터**를 인용했음.
> 아래 4개 문제를 발견·수정하고, 재발 방지용 **자동 검증 하네스**를 추가했다.

## 발견된 문제 (Before)

| # | 문제 | 심각도 | 근거 |
|---|---|---|---|
| 1 | **법적 근거가 전부 제1조(목적)** — "이 조례는 …을 목적으로 한다"만 인용. 정작 사용자 질문에 해당하는 실효 조문(수수료·배출방법·보조금)은 미연결 → 그라운딩이 장식에 불과 | 높음 | 7개 링크 전부 `제1조(목적)` |
| 2 | **쓰레기(오류) 데이터를 법적 근거로 인용** — 지원기관이 `{"Law":"일치하는 자치법규가 없습니다…"}` 청크를 "법적 근거"로 표시 | **치명** | `ord-1987025-full` |
| 3 | **L2 코퍼스에 수집 실패 청크 79건 혼입** — 검색·인용 시 헛소리 노출 가능 | 높음 | 전수 스캔 80건(오류 응답) |
| 4 | **무결성 검증 부재** — 끊긴 ID·보일러플레이트·잘못된 참조를 막는 자동 검사 없음 | 중간 | — |

## 개선 (After)

1. **실효 조문 동적 선택** (`build_ontology.py` + `ontology.json:article_keywords`)
   - 목적/정의/장제목/오류 청크를 제외하고, 질문 관련 키워드로 **실효 조문**을 점수화해 선택.
   - 예) "대형폐기물 수수료" → **제25조(수수료 부과·징수)**, "임금" → **노동기본조례 제6조(권리 보호 지원)**.
   - 백엔드는 응답 시 도메인 법적근거 중 **질문 토큰과 가장 겹치는 조문**을 상위 2건 인용(질문별 동적).
2. **쓰레기/보일러플레이트 차단**
   - 빌드·검증·**런타임 검색**(`backend.valid_ord_content`)에서 오류 청크 제외 → L2 11,268→**11,189**(79건 정화).
   - 가족센터 조례(수집 실패)는 법적 근거에서 자동 제외(헛링크 0).
3. **무결성 검증 하네스** (`scripts/validate_ontology.py`)
   - legal_basis ID 실재·실효성, faq_id 실재, prerequisites/relations 참조 유효성, 필수 메타, 지원정책 형식 검사.
   - **현재: 오류 0 / 경고 0.** CI 게이트(종료코드)로 재발 방지.
4. **AX 가치 추가**
   - 도메인 온톨로지로 **자격·선행절차**(예: 건강보험 ← "외국인등록 먼저") + **지원정책(혜택) 카드**(간이대지급금·다누리콜센터·보육료 지원 등) 자동 노출.

## 검증 결과 (재현 가능)

```
python scripts/build_ontology.py       # 실효조문 12건 링크, 수집실패 1건 제외
python scripts/validate_ontology.py    # ✅ 오류 0 / 경고 0
python scripts/eval.py                 # 라우팅·인용·환각거부 전 지표 통과
curl /api/health                       # l2_dropped_garbage: 79, ontology_legal_links: 12
```

## 남은 한계 (정직한 기록)

- 법적 근거는 **화성시 자치법규(L2)** 한정. 건강보험·운전면허·행정은 **국가법** 영역이라 자치법규 링크가 없고 `national_law` 텍스트로만 표기(헛링크를 만들지 않기 위함).
- 조문 선택은 키워드 기반. 실동작(Ollama+BGE-M3) 전환 시 임베딩으로 조문 단위 정밀 매칭을 확대할 수 있음.
- 온톨로지는 현재 도메인 단위 그래프. 향후 엔티티(서류·자격·기관) 노드 단위로 확장 가능.
