# 실제 공식 서식(.hwpx) 등록 — 빈칸 자동 채움

화성ON은 **한글 없이 웹 서버에서 실제 공식 .hwpx 서식의 빈칸을 채워** 내려줍니다.
(검증: 서버가 채운 실제 .hwpx가 진짜 한글에서 정상적으로 열리고 채운 값이 인식됨.)

## 동작 방식
- `kb/forms/registry.json` 에 도메인별로 **서식 파일 + 치환맵(예시문구 → 사용자 필드)** 을 등록.
- 사용자가 폼을 채우면 `/api/draft/hwpx` 가 그 **실제 서식**의 해당 문구를 사용자 값으로 바꿔 내려줌.
- 등록이 없으면 검증된 **생성본 .hwpx** 로 자동 폴백(서비스 항상 동작).

## 추가 절차
1. **공식 서식을 .hwpx로 확보**
   - 정부·지자체는 다수 서식을 .hwpx로 배포(예: 정부24, 노동포털, 각 부처 서식자료실).
   - .hwp(구형 바이너리)만 있으면 **한 번만 .hwpx로 변환**:
     무료 한컴독스(웹)·폴라리스(hwp.polarisoffice.com)에서 열어 "다른 이름으로 저장 → HWPX",
     또는 데스크톱 한글에서 저장.
2. 변환한 파일을 `kb/forms/` 에 넣고, `registry.json` 에 등록:
   ```json
   {
     "노무": {
       "template": "jinjeong_imgeum.hwpx",
       "replace": {
         "홍 길 동": "{name}",
         "한국회사(주)": "{company}",
         "3,000,000원": "{amount_won}",
         "880808-1234567": "{rrn}"
       }
     }
   }
   ```
   - 왼쪽 = **서식에 들어있는 예시문구(또는 빈칸 표시 토큰)**, 오른쪽 = **사용자 필드**(`FORM_SCHEMAS`의 key).
   - 특수 필드: `{amount_won}`(천단위+원), `{amount}`(천단위).
3. 서버 재시작 → 해당 도메인은 **실제 공식 서식**으로 채워 내려갑니다.

## 팁(정확도)
- 치환은 한 `<hp:t>` 텍스트 노드 단위로 동작하므로, 예시문구가 서식 안에서 **한 덩어리**여야 안전.
  (셀이 빈칸이면 그 셀에 `{{성명}}` 같은 토큰을 한 번 넣어두고 등록하면 가장 확실.)
- 등록 전 `python scripts/verify_form.py <파일.hwpx>` 로 구조 유효성(있다면) 점검 권장.
